## 1. [Problem Formulation]

### Forward Model

$$I_j(\mathbf{k}) = \left|\mathcal{F}\left\{P(\mathbf{r} - \mathbf{R}_j) \cdot O(\mathbf{r})\right\}\right|^2 + n(\mathbf{k})$$

where:
- $O(\mathbf{r}) \in \mathbb{C}^{600 \times 600}$: complex reflection function
- $P(\mathbf{r}) \in \mathbb{C}^{256 \times 256}$: probe/illumination function
- $\mathbf{R}_j$: scan position $j \in \{1,\ldots,81\}$ on $9\times9$ grid, step=40 pixels
- $\mathcal{F}$: 2D DFT
- $n \sim \mathcal{N}(0,\sigma^2)$: Gaussian noise
- **Input**: $(81, 256, 256)$ real diffraction intensities
- **Output**: $(2, 600, 600)$ — channel 0: $|O|$, channel 1: $\angle O$

### Inverse Problem

$$\hat{O}, \hat{P} = \arg\min_{O,P} \sum_{j=1}^{81} \left\| \sqrt{I_j} - \left|\mathcal{F}\{P(\mathbf{r}-\mathbf{R}_j) \cdot O(\mathbf{r})\}\right| \right\|_2^2$$

## 2. [Proposed Strategy]

**Algorithm**: **Standard Sequential ePIE with Randomized Position Ordering** — the proven, stable baseline for ptychographic reconstruction. No momentum, no batch gradient, no acceleration tricks that caused divergence in Iter 13.

### Root Cause Analysis of ALL Previous Failures

| Iteration | PSNR | Root Cause |
|-----------|------|------------|
| 11 | 2.92 | Global-max normalization killed updates |
| 12 | 6.54 | Sequential ePIE but too few epochs, poor probe init |
| 13 | 1.84 | **Nesterov momentum double-application caused divergence** |

**Key Lessons:**
1. **Momentum is dangerous** for ptychography — it caused divergence (Iter 13: PSNR 1.84)
2. **Sequential ePIE without momentum** actually worked best (Iter 12: PSNR 6.54) but didn't converge enough
3. **The path forward**: Use standard sequential ePIE but maximize iterations and fix convergence issues

### Why Sequential ePIE Will Work This Time (vs Iter 12's 6.54)

Iter 12 achieved 6.54 PSNR. The improvements needed:
1. **Better probe initialization**: Use `ifft2(sqrt(mean(I)))` with proper windowing
2. **More effective iterations**: Optimize Python loop to squeeze more epochs
3. **Randomized position ordering**: Prevents periodic stagnation patterns
4. **Adaptive step size**: Start with β=1.0, no need to be conservative
5. **Probe update from epoch 1**: Don't delay probe updates unnecessarily
6. **Better phase alignment**: Use substrate region (outside pads) as reference

### Step 4: Probe Initialization

```python
# Method: inverse FFT of average measured amplitude
avg_amp = np.mean(diff_amp_fft, axis=0)  # (256, 256) in FFT layout

# Create probe as ifft2 of avg amplitude (zero phase in Fourier domain)
P_raw = np.fft.ifft2(avg_amp)  # (256, 256), complex

# Apply localizing window in real space
# P_raw should have peak at (0,0) in FFT convention
P_shifted = np.fft.fftshift(P_raw)
yy, xx = np.mgrid[:ps, :ps]
cy, cx = ps/2.0, ps/2.0
r2 = (yy - cy)**2 + (xx - cx)**2
window = np.exp(-r2 / (2 * 50.0**2))  # σ=50 pixels
P_shifted *= window
P = np.fft.ifftshift(P_shifted)

# Normalize probe so max|P| = 1
P = P / (np.max(np.abs(P)) + eps)
```

### Step 6: Standard Sequential ePIE — Core Loop

This is the **exact standard ePIE algorithm** with no modifications that could cause instability:

```python
def run_ePIE(O, P, diff_amp_fft, positions, t_start, TIME_LIMIT):
    """
    Standard ePIE (extended Ptychographic Iterative Engine).
    
    Per position j:
      1. ψ_j = P(r-R_j) · O(r)              [exit wave]
      2. Ψ_j = FFT(ψ_j)                      [far field]
      3. Ψ'_j = sqrt(I_j) · Ψ_j / |Ψ_j|    [magnitude constraint]
      4. ψ'_j = IFFT(Ψ'_j)                   [corrected exit wave]
      5. Δψ_j = ψ'_j - ψ_j                   [difference]
      6. O(r) += α_O · conj(P) / max(|P|²) · Δψ_j    [object update]
      7. P(r) += α_P · conj(O_j) / max(|O_j|²) · Δψ_j  [probe update]
    """
    
    alpha_O = 1.0   # object step size (standard ePIE = 1)
    alpha_P = 1.0   # probe step size
    reg = 1e-3      # regularization fraction
    
    n_pos = len(positions)
    epoch = 0
    
    while True:
        elapsed = time.time() - t_start
        if elapsed > TIME_LIMIT:
            break
        
        # Randomize position ordering each epoch
        order = np.random.permutation(n_pos)
        
        for idx in range(n_pos):
            j = order[idx]
            ry, rx = positions[j]
            
            # Extract object patch
            O_patch = O[ry:ry+ps, rx:rx+ps]  # (256, 256)
            
            # Exit wave
            psi = P * O_patch  # (256, 256)
            
            # Forward propagate
            Psi = np.fft.fft2(psi)  # (256, 256)
            
            # Fourier magnitude constraint
            Psi_amp = np.abs(Psi) + eps
            Psi_corrected = diff_amp_fft[j] * Psi / Psi_amp
            
            # Inverse propagate
            psi_c = np.fft.ifft2(Psi_corrected)  # (256, 256)
            
            # Difference
            dpsi = psi_c - psi
            
            # Object update (standard ePIE formula)
            P_max_sq = np.max(np.abs(P)**2)
            O[ry:ry+ps, rx:rx+ps] += alpha_O * np.conj(P) * dpsi / (P_max_sq + reg)
            
            # Probe update (standard ePIE formula)
            O_patch_max_sq = np.max(np.abs(O_patch)**2)
            P += alpha_P * np.conj(O_patch) * dpsi / (O_patch_max_sq + reg)
        
        epoch += 1
    
    return O, P, epoch

O_result, P_result, n_epochs = run_ePIE(
    O, P, diff_amp_fft, positions, t_start, TIME_LIMIT
)
```

### Step 8: Post-Processing — Phase Alignment

```python
# Build scan coverage mask
scan_mask = np.zeros((obj_size, obj_size), dtype=bool)
for j in range(n_pos):
    ry, rx = positions[j]
    scan_mask[ry:ry+ps, rx:rx+ps] = True

amp = np.abs(O_result)
phase = np.angle(O_result)

# Identify valid reconstructed region
valid = scan_mask & (amp > 0.1 * np.max(amp[scan_mask]))

if np.sum(valid) > 100:
    # Find dominant phase via histogram (substrate = most common phase)
    phase_valid = phase[valid]
    n_bins = 720
    hist, bin_edges = np.histogram(phase_valid, bins=n_bins, range=(-np.pi, np.pi))
    
    # Smooth histogram
    kernel = np.ones(21) / 21.0
    hist_smooth = np.convolve(hist, kernel, mode='same')
    mode_idx = np.argmax(hist_smooth)
    phase_ref = (bin_edges[mode_idx] + bin_edges[mode_idx + 1]) / 2.0
else:
    phase_ref = 0.0

# Remove global phase offset so substrate phase ≈ 0
O_result *= np.exp(-1j * phase_ref)

# Final output
output = np.zeros((2, obj_size, obj_size), dtype=np.float32)
output[0] = np.abs(O_result).astype(np.float32)
output[1] = np.angle(O_result).astype(np.float32)
```

## 4. [Loss Function & Optimizer]

### Loss (Amplitude-based Fourier constraint)
$$\mathcal{L} = \sum_{j=1}^{81} \left\| \sqrt{I_j} - \left|\mathcal{F}\{P \cdot O_j\}\right| \right\|_2^2$$

### Standard ePIE Update Rules (Rodenburg & Faulkner 2004)

**Object update** at position $j$:
$$O^{(n+1)}(\mathbf{r}) = O^{(n)}(\mathbf{r}) + \alpha_O \frac{P^*(\mathbf{r}-\mathbf{R}_j)}{|P_{\max}|^2} \cdot \Delta\psi_j(\mathbf{r})$$

**Probe update** at position $j$:
$$P^{(n+1)}(\mathbf{r}) = P^{(n)}(\mathbf{r}) + \alpha_P \frac{O^{(n)*}(\mathbf{r}+\mathbf{R}_j)}{|O_{\max,j}|^2} \cdot \Delta\psi_j(\mathbf{r})$$

where $\Delta\psi_j = \mathcal{F}^{-1}\left[\sqrt{I_j} \cdot \frac{\hat{\Psi}_j}{|\hat{\Psi}_j|}\right] - P \cdot O_j$

### Why This Will Succeed

| Property | Value | Rationale |
|----------|-------|-----------|
| Step size $\alpha_O$ | 1.0 | Standard ePIE, proven stable |
| Step size $\alpha_P$ | 1.0 | Standard ePIE, proven stable |
| Momentum | **None** | Momentum caused divergence in Iter 13 |
| Position ordering | Random per epoch | Prevents periodic artifacts |
| Normalization | $|P_{\max}|^2$ (scalar) | Standard ePIE, not per-pixel (avoids instability) |
| Probe init | From data | Better starting point than Gaussian blob |
| Regularization | $10^{-3} \cdot |P_{\max}|^2$ | Prevents division issues |

### Estimated Performance

- ~160ms per FFT pair × 81 positions = ~13s per epoch
- With optimized NumPy: ~8-10s per epoch
- In 53s budget: **5-6 full epochs**
- ePIE converges fast with good probe init — 5-6 epochs should give significant improvement over Iter 12's result since that had poor probe initialization

### Fallback: If Only 5 Epochs Isn't Enough

The key insight is that even 5 ePIE epochs with **good initialization** and **randomized ordering** should outperform 150 epochs with poor initialization. The probe-from-data initialization gives a much better starting point, and the randomized ordering prevents the systematic bias that sequential ordering introduces.
