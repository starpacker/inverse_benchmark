## 1. [Problem Formulation]

### Forward Model

$$I_j(\mathbf{k}) = \left|\mathcal{F}\left\{P(\mathbf{r} - \mathbf{R}_j) \cdot O(\mathbf{r})\right\}\right|^2 + n(\mathbf{k})$$

where:
- $O(\mathbf{r}) \in \mathbb{C}^{600 \times 600}$: complex reflection function
- $P(\mathbf{r}) \in \mathbb{C}^{256 \times 256}$: probe/illumination function
- $\mathbf{R}_j$: scan position $j \in \{1, \ldots, 81\}$ on a $9 \times 9$ grid, step = 40 pixels
- $\mathcal{F}$: 2D FFT (numpy convention)
- $n \sim \mathcal{N}(0, \sigma^2)$: additive Gaussian noise
- **Input**: $(81, 256, 256)$ diffraction intensities
- **Output**: $(2, 600, 600)$ — channel 0: amplitude $|O|$, channel 1: phase $\angle O$

### Physical Parameters
- $\lambda = 13.5$ nm, pixel size 27 nm, pad size 22 px, pitch 30 px, 15×15 array
- Recession depth $h \in [1,3]$ nm → $\Delta\Phi = 4\pi h/\lambda \in [0.93, 2.79]$ rad
- EUV reflectivities: $|R_{\text{sub}}| \approx 0.011$, $|R_{\text{Cu}}| \approx 0.037$ (very weak signals)

## 2. [Proposed Strategy]

**Algorithm**: Vectorized/batched ePIE with rpPIE reflectivity prior, **≤60 epochs**, exploiting batch FFT operations to stay within time budget.

### Step 4: Initialization

#### 4.1 FFT Convention Detection (Quick, One-Shot)

Before iterating, determine if input diffraction patterns have DC at center or corner:

```python
# Test: DC at center means the brightest pixel is near (128,128)
center_energy = np.sum(I[0, 112:144, 112:144])
corner_energy = np.sum(I[0, :32, :32]) + np.sum(I[0, :32, -32:]) + \

use_shift = (center_energy > corner_energy)
```

If `use_shift=True`, pre-convert all measured amplitudes to FFT-order once:
```python

else:

```

This avoids per-position shifting inside the loop.

#### 4.2 Object Initialization

### Step 5: Batched ePIE/rpPIE Solver

#### 5.1 Batch Patch Extraction

```python
def extract_patches(O, positions, ps):
    """Extract all 81 patches from O. Returns (81, 256, 256) complex array."""
    patches = np.empty((len(positions), ps, ps), dtype=O.dtype)
    for k, (ry, rx) in enumerate(positions):
        patches[k] = O[ry:ry+ps, rx:rx+ps]
    return patches
```

#### 5.2 Batch Scatter-Add Update

```python
def scatter_add(O, positions, ps, updates):
    """Add (81, 256, 256) updates back to O at respective positions."""
    # Use accumulator to handle overlapping regions correctly
    for k, (ry, rx) in enumerate(positions):
        O[ry:ry+ps, rx:rx+ps] += updates[k]
```

**Note**: Overlapping regions get summed contributions from multiple positions. This is the correct ePIE behavior when processing all positions before applying updates (simultaneous update mode). However, standard ePIE is sequential. 

**Compromise for speed**: Use **mini-batch sequential** — process positions in random order but batch the FFT computations. Actually, the simplest performant approach: **process all 81 sequentially but use optimized numpy operations within each position**. The bottleneck is the FFT, and numpy's fft2 on 256×256 is fast (~1ms). 81 × 2 FFTs × 60 epochs = 9,720 FFTs ≈ 10 seconds.

**Even faster**: True batch approach with simultaneous update. Process all 81 at once:

```python
for epoch in range(n_epochs):
    # 1. Extract all patches: (81, 256, 256)
    patches = extract_patches(O, positions, ps)
    
    # 2. Compute exit waves: (81, 256, 256)
    psi = P[np.newaxis, :, :] * patches  # broadcasting
    
    # 3. Batch forward FFT: (81, 256, 256)
    Psi = np.fft.fft2(psi)  # batched over axis 0
    
    # 4. Magnitude replacement
    Psi_amp = np.abs(Psi)
    Psi_c = diff_amp_fft * Psi / (Psi_amp + eps)
    
    # 5. Batch inverse FFT
    psi_c = np.fft.ifft2(Psi_c)
    
    # 6. Exit wave differences: (81, 256, 256)
    dpsi = psi_c - psi
    
    # 7. Object update (simultaneous ePIE)
    P_max_sq = np.max(np.abs(P)**2)
    obj_updates = alpha_O * np.conj(P)[np.newaxis, :, :] * dpsi / (P_max_sq + eps)
    
    # Accumulate updates with overlap normalization
    O_update_num = np.zeros_like(O)
    O_update_den = np.zeros((obj_size, obj_size), dtype=np.float64)
    for k, (ry, rx) in enumerate(positions):
        O_update_num[ry:ry+ps, rx:rx+ps] += obj_updates[k]
        O_update_den[ry:ry+ps, rx:rx+ps] += 1.0
    
    O_update_den = np.maximum(O_update_den, 1.0)
    O += O_update_num / O_update_den
    
    # 8. Probe update (average over all positions)
    if epoch >= 1:
        O_max_sq = np.max(np.abs(patches)**2)
        probe_updates = alpha_P * np.conj(patches) * dpsi / (O_max_sq + eps)
        P += np.mean(probe_updates, axis=0)
    
    # 9. Constraints
    amp = np.abs(O)
    phase = np.angle(O)
    O = np.minimum(amp, 1.0) * np.exp(1j * phase)
    
    # 10. Reflectivity prior
    if epoch >= 1:
        gamma = min(1.0, epoch / 5.0)
        beta_prior = 0.7
        blend = gamma * (1 - beta_prior)
        O[~S] = (1 - blend) * O[~S] + blend * R_p
```

**Time estimate for batch approach**:
- `extract_patches`: 81 array slices → ~0.5ms
- `P * patches` broadcast: (256,256) × (81,256,256) → ~5ms  
- `fft2` on (81,256,256): ~50ms
- `ifft2` on (81,256,256): ~50ms
- Scatter accumulation: 81 additions → ~5ms
- Constraints: element-wise on (600,600) → ~1ms
- **Total per epoch**: ~120ms
- **60 epochs**: ~7.2s → **well within budget**

#### 5.3 Hyperparameters

```python
n_epochs = 60
alpha_O = 1.0       # object step size
alpha_P = 0.5       # probe step size (conservative, probe is shared)
eps = 1e-10         # numerical stability
beta_prior = 0.7    # reflectivity prior strength
```

#### 5.4 Refinement: Sequential Mini-Batches (Hybrid)

The fully simultaneous approach may converge slower than sequential ePIE. A compromise: **split 81 positions into ~9 mini-batches of 9**, process each batch simultaneously, update O between batches. This gives 9 sequential updates per epoch (better convergence) with batch FFT speedup:

```python
batch_size = 9  # or 27
n_batches = 81 // batch_size

for epoch in range(n_epochs):
    order = np.random.permutation(81)
    
    for b in range(n_batches):
        batch_idx = order[b*batch_size:(b+1)*batch_size]
        batch_pos = positions[batch_idx]
        
        # Extract patches for this batch
        patches = np.stack([O[ry:ry+ps, rx:rx+ps] for ry, rx in batch_pos])
        
        # Batch FFT forward
        psi = P[None] * patches
        Psi = np.fft.fft2(psi)
        Psi_amp = np.abs(Psi)
        Psi_c = diff_amp_fft[batch_idx] * Psi / (Psi_amp + eps)
        psi_c = np.fft.ifft2(Psi_c)
        dpsi = psi_c - psi
        
        # Object update
        P_max_sq = np.max(np.abs(P)**2)
        obj_upd = alpha_O * np.conj(P)[None] * dpsi / (P_max_sq + eps)
        
        num = np.zeros_like(O)
        den = np.zeros((obj_size, obj_size), dtype=np.float64)
        for i, (ry, rx) in enumerate(batch_pos):
            num[ry:ry+ps, rx:rx+ps] += obj_upd[i]
            den[ry:ry+ps, rx:rx+ps] += 1.0
        den = np.maximum(den, 1.0)
        O += num / den
        
        # Probe update
        if epoch >= 1:
            O_max_sq = np.max(np.abs(patches)**2)
            P += alpha_P * np.mean(np.conj(patches) * dpsi, axis=0) / (O_max_sq + eps)
    
    # End-of-epoch constraints
    amp = np.abs(O)
    O = np.minimum(amp, 1.0) * np.exp(1j * np.angle(O))
    
    if epoch >= 1:
        gamma = min(1.0, epoch / 5.0)
        blend = gamma * 0.3  # (1 - beta_prior) = 0.3
        O[~S] = (1 - blend) * O[~S] + blend * R_p
```

**Time**: 9 batches × ~15ms each = ~135ms/epoch × 60 = ~8s. Safe.

**Decision**: Use **batch_size = 9** (9 batches of 9) as the best speed/convergence tradeoff. Each batch has well-separated positions (due to random shuffling), minimizing overlap conflicts within a batch.

### Step 6: Output Formatting

```python
# Remove global phase ambiguity
substrate_vals = O[~S]
phase_ref = np.angle(np.mean(substrate_vals))
O *= np.exp(-1j * phase_ref)

output = np.zeros((2, 600, 600), dtype=np.float32)
output[0] = np.abs(O).astype(np.float32)
output[1] = np.angle(O).astype(np.float32)
return output
```

### Step 7: Loss Function & Optimizer

**Internal loss** (not computed during run to save time, only conceptual):
$$\mathcal{E} = \frac{1}{J}\sum_j \frac{\||\mathcal{F}\{P \cdot O_j\}| - \sqrt{I_j}\|^2}{\|\sqrt{I_j}\|^2}$$

**Optimizer**: ePIE is a coordinate-descent-like algorithm. Step sizes $\alpha_O = 1.0$, $\alpha_P = 0.5$ are standard choices.

### Step 8: Robustness Measures

1. **FFT convention**: Detected once at initialization (no wasted epochs)
2. **Probe energy normalization**: After each epoch, if $\|P\|$ grows > 5× initial, renormalize
3. **No convergence monitoring loop** (saves time)
4. **No duplicate code in `__main__`** — single call to `solver.solve()`
5. **eps = 1e-10** throughout for numerical safety
