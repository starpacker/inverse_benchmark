

# Revised Plan: Coherent Diffractive Imaging Phase Retrieval (Iteration 4)

## [Problem Formulation]

### Hard Constraints
- **Input**: `(81, 256, 256)` — 81 ptychographic diffraction intensity patterns, each 256×256
- **Output**: `(2, 600, 600)` — channel 0: magnitude |O(r)|, channel 1: phase arg(O(r))

### Forward Model

$$I_j(\mathbf{k}) = \left|\mathcal{F}\left\{P(\mathbf{r} - \mathbf{R}_j) \cdot O(\mathbf{r})\right\}\right|^2 + n, \quad j = 1, \ldots, 81$$

- $O(\mathbf{r}) \in \mathbb{C}^{600 \times 600}$: complex object reflection function
- $P(\mathbf{r}) \in \mathbb{C}^{256 \times 256}$: illumination probe
- $\mathbf{R}_j$: scan positions on a 9×9 raster grid, step = 43 pixels (since $(600-256)/8 = 43$)
- $I_j \in \mathbb{R}_{\geq 0}^{256 \times 256}$: measured far-field intensity
- $n$: additive Gaussian noise

### Inverse Problem

Recover $O(\mathbf{r})$ (and optionally $P(\mathbf{r})$) from intensity-only measurements $\{I_j\}_{j=1}^{81}$.

### Root Cause Analysis of Previous Failures

**Iteration 2** (error 12.4): Hardcoded reflectivity priors with wrong TMM constants → completely wrong reconstruction.

**Iteration 3** (PSNR 6.54, SSIM 0.22): Standard ePIE with:
- **Wrong FFT convention** — the `ifftshift→fft2→fftshift` pattern is likely incorrect
- Insufficient iterations (300)
- Poor probe initialization and early probe updates destabilizing convergence
- Regularization too small

**The FFT convention is the #1 suspect.** If the forward/backward transforms are wrong, every modulus replacement corrupts the exit wave, and the algorithm diverges rather than converges.

---

## [Proposed Strategy]

### Algorithm: rPIE (regularized PIE) with careful FFT handling and multiple convention testing

**Key changes from previous attempt:**

1. **FFT Convention**: Try ALL FOUR possible conventions systematically, pick the one with lowest Fourier error after 10 iterations. The four conventions are:
   - `fft2(x)` / `ifft2(x)` (no shifts)
   - `fftshift(fft2(x))` / `ifft2(ifftshift(x))`
   - `fftshift(fft2(ifftshift(x)))` / `fftshift(ifft2(ifftshift(x)))`
   - `fft2(fftshift(x))` / `ifftshift(ifft2(x))`

2. **rPIE update** (more robust than ePIE): Uses a weighted denominator instead of max-based normalization.

3. **Delayed probe update**: Start probe recovery only after epoch 50 (let object stabilize first).

4. **More iterations**: 800 epochs.

5. **Better probe initialization**: Use the data itself to estimate probe power, and try both Gaussian and flat initializations.

### Step 2: FFT Convention Determination (CRITICAL)

**This is the single most important step.** We systematically test conventions.

Define four forward/backward pairs:

```python
# Convention 0: bare FFT (DC at corner)
def fwd_0(x): return np.fft.fft2(x)
def bwd_0(x): return np.fft.ifft2(x)

# Convention 1: fftshift output (DC at center in Fourier domain)
def fwd_1(x): return np.fft.fftshift(np.fft.fft2(x))
def bwd_1(x): return np.fft.ifft2(np.fft.ifftshift(x))

# Convention 2: ifftshift input, fftshift output (centered both domains)
def fwd_2(x): return np.fft.fftshift(np.fft.fft2(np.fft.ifftshift(x)))
def bwd_2(x): return np.fft.fftshift(np.fft.ifft2(np.fft.ifftshift(x)))

# Convention 3: fftshift input (object centered, DC at corner)
def fwd_3(x): return np.fft.fft2(np.fft.fftshift(x))
def bwd_3(x): return np.fft.ifftshift(np.fft.ifft2(x))
```

**Convention selection protocol:**
```
For each convention c in {0, 1, 2, 3}:
    Initialize O = ones(600,600), P = gaussian_probe(256,256)
    Run 10 ePIE iterations with convention c
    Compute Fourier error: E_c = mean over j of ||abs(fwd(P*O_patch_j)) - amplitudes[j]||^2 / ||amplitudes[j]||^2
    
Select c* = argmin_c E_c
```

**Rationale:** The wrong FFT convention will produce large Fourier errors because the modulus replacement will be applied to incorrectly-positioned frequency components, and the back-propagated correction will be spatially scrambled. The correct convention will show rapid error decrease.

**Expected outcome:** For centered diffraction data (DC at pixel 128,128), Convention 1 or 2 should win. For corner-DC data, Convention 0 or 3.

### Step 3: rPIE Algorithm (Main Solver)

**3.1 rPIE Update Equations (Maiden et al., 2017):**

The rPIE update uses a regularized denominator that is more stable than ePIE's max-based normalization:

**Object update:**
$$O_{n+1}(\mathbf{r}) = O_n(\mathbf{r}) + \frac{|P(\mathbf{r}-\mathbf{R}_j)|^{\gamma_O} \cdot P^*(\mathbf{r}-\mathbf{R}_j)}{(1-\alpha_O)|P(\mathbf{r}-\mathbf{R}_j)|^2 + \alpha_O \cdot \max_\mathbf{r}|P|^2} \cdot \Delta\Psi_j(\mathbf{r})$$

**Probe update (when active):**
$$P_{n+1}(\mathbf{r}) = P_n(\mathbf{r}) + \frac{|O(\mathbf{r}+\mathbf{R}_j)|^{\gamma_P} \cdot O^*(\mathbf{r}+\mathbf{R}_j)}{(1-\alpha_P)|O(\mathbf{r}+\mathbf{R}_j)|^2 + \alpha_P \cdot \max_\mathbf{r}|O|^2} \cdot \Delta\Psi_j(\mathbf{r})$$

where:
- $\Delta\Psi_j = \Psi_{c,j} - \Psi_j$ (difference between corrected and current exit wave)
- $\gamma_O = \gamma_P = 1$ (standard rPIE, equivalent to setting the weighting exponent)
- $\alpha_O = \alpha_P = 0.1$ (interpolation between ePIE-like max and PIE-like local denominator)
- When $\alpha = 1$: reduces to ePIE; when $\alpha = 0$: reduces to PIE

**Simplified implementation (using $\gamma=0$ for robustness, which gives the standard PIE-like form):**

```python
# Object update
denom_O = (1 - alpha_O) * abs(P)**2 + alpha_O * P_max_sq
O_patch += conj(P) / denom_O * delta_psi

# Probe update  
denom_P = (1 - alpha_P) * abs(O_patch)**2 + alpha_P * O_max_sq
P += conj(O_patch_old) / denom_P * delta_psi
```

**3.2 Modulus replacement (Fourier constraint):**
```python
Psi_hat = forward(psi)  # using selected convention
mag = np.abs(Psi_hat)
eps = 1e-8  # FIXED small epsilon, not scaled
Psi_hat_corrected = amplitudes[j] * Psi_hat / (mag + eps)
psi_corrected = backward(Psi_hat_corrected)
delta_psi = psi_corrected - psi
```

**CRITICAL:** The `eps` should be a small fixed constant (1e-8), NOT scaled by max(mag). Scaling by max creates a large epsilon that corrupts the phase of weak Fourier components.

**3.3 Initialization:**

```python
# Object: uniform magnitude, zero phase
O = np.ones((600, 600), dtype=np.complex128)

# Probe: Gaussian with radius calibrated from data
# Estimate probe power from total diffracted intensity (Parseval's theorem)
mean_total_intensity = np.mean(np.sum(data, axis=(1,2)))
# |FFT(P*O)|^2 summed = N^2 * sum(|P*O|^2), so sum(|P*O|^2) ≈ mean_total_intensity / N^2
# With O≈1, sum(|P|^2) ≈ mean_total_intensity / (256^2)

Y, X = np.mgrid[-128:128, -128:128]
R = np.sqrt(X**2 + Y**2)
sigma_probe = 60  # pixels, ~half of probe window
P = np.exp(-R**2 / (2 * sigma_probe**2)).astype(np.complex128)

# Scale probe to match measured intensity
probe_power = mean_total_intensity / (256**2)  # Parseval normalization
P *= np.sqrt(probe_power / np.sum(np.abs(P)**2))
```

**3.4 Main Loop:**

```python
N_iter = 800
alpha_O = 0.1  # rPIE regularization for object
alpha_P = 0.1  # rPIE regularization for probe
probe_update_start = 50  # delay probe updates

for epoch in range(N_iter):
    # Shuffle scan order for each epoch
    order = np.random.permutation(81)
    
    for j in order:
        ry, rx = positions[j]
        
        # Extract patch
        O_patch = O[ry:ry+256, rx:rx+256].copy()
        
        # Exit wave
        psi = P * O_patch
        
        # Forward propagate (selected convention)
        Psi_hat = forward(psi)
        
        # Modulus replacement
        mag = np.abs(Psi_hat)
        Psi_hat_c = amplitudes[j] * Psi_hat / (mag + 1e-8)
        
        # Back-propagate
        psi_c = backward(Psi_hat_c)
        
        # Update difference
        dpsi = psi_c - psi
        
        # Object update (rPIE)
        P_abs_sq = np.abs(P)**2
        P_max_sq = np.max(P_abs_sq)
        denom_O = (1 - alpha_O) * P_abs_sq + alpha_O * P_max_sq
        O[ry:ry+256, rx:rx+256] += np.conj(P) / denom_O * dpsi
        
        # Probe update (rPIE, delayed start)
        if epoch >= probe_update_start:
            O_abs_sq = np.abs(O_patch)**2
            O_max_sq = np.max(O_abs_sq)
            denom_P = (1 - alpha_P) * O_abs_sq + alpha_P * O_max_sq
            P += np.conj(O_patch) / denom_P * dpsi
    
    # Monitor Fourier error every 50 epochs
    if epoch % 50 == 0:
        compute_and_print_fourier_error()
```

**3.5 Hyperparameter schedule (adaptive):**
```python
# After epoch 200, tighten regularization for finer updates

    alpha_O = 0.05
    alpha_P = 0.05

# After epoch 500, very fine updates
if epoch == 500:
    alpha_O = 0.02
    alpha_P = 0.02
```

### Step 4: Post-Processing

**4.1 Global phase ambiguity removal:**

Ptychography with probe recovery resolves most ambiguities, but a global phase offset remains.

```python
# Strategy: set the phase of the brightest region (or background) as reference
# Find the most common phase value in high-magnitude regions
high_mag_mask = np.abs(O) > 0.5 * np.max(np.abs(O))
if np.any(high_mag_mask):
    ref_phase = np.angle(np.mean(O[high_mag_mask]))
    O *= np.exp(-1j * ref_phase)
```

**4.2 Flip/rotation ambiguity check:**

Ptychography can sometimes produce a conjugate/flipped solution. We should check if the output needs conjugation:

### Step 5: Robustness and Fallback Strategies

**5.1 If FFT convention test is inconclusive (errors similar for multiple conventions):**
- Run full 200 iterations with the top-2 conventions
- Pick the one with lower final Fourier error

**5.2 If reconstruction shows periodic artifacts:**
- The scan step (43) may be slightly off
- Try step = 42 or 44 and check if Fourier error improves
- Recompute: step=42 → obj_size = 42×8+256 = 592 (need padding to 600)
- step=43 → obj_size = 43×8+256 = 600 ✓ (exact)
- Stick with 43

**5.3 If probe update destabilizes:**
- Increase `probe_update_start` to 100 or 150
- Reduce probe update strength: multiply probe update by 0.5
- Or fix probe entirely (no blind recovery) if a good initial guess works

**5.4 Alternative probe initializations to try (in order):**
1. Gaussian σ=60
2. Gaussian σ=40 (tighter)
3. Flat disk radius=80 with soft edge
4. Use autocorrelation of average diffraction pattern to estimate probe support

**5.5 If PSNR is still below baseline after 800 iterations:**
- Increase to 2000 iterations
- Try momentum-accelerated PIE (mPIE)
- Try position correction (annealing PIE)
