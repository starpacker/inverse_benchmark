# Ablation Study & Troubleshooting Guide

## Ablation Study Overview
We have introduced `pipeline_ablation.py` to test the coding capability of the agent in isolation, without the variability of the Planning Phase.

### Key Differences from Main Pipeline
1.  **Golden Plan Usage**: Instead of generating a plan using LLM, the pipeline directly loads the `golden_plan.json` generated during Phase 0 (Dataset Construction).
2.  **No Backtracking**: If execution or verification fails, the agent is forced to retry in the **CODING** phase. It will never revert to the PLANNING phase, even if the error seems to be a logical/planning error.
3.  **Focus**: This tests purely the `CodeGenerator` and `CodeExecutor`'s ability to translate a perfect plan into working code.

### Running the Ablation Study
```bash
python pipeline_ablation.py \
  --task_name "your_task_name" \
  --gt_code "/path/to/ground_truth.py" \
  --working_folder "/path/to/sandbox" \
  --output_dir "results_ablation"
```

## Artifact Locations & Troubleshooting
When code generation fails, check the following locations to diagnose the issue.

### 1. Dataset Artifacts (Phase 0)
Located in: `/data/yjh/benchmark_dataset/{task_name}/` (or as configured in `settings.py`)
*   **`task_description.md`**: The problem statement given to the agent. Check if it's clear.
*   **`golden_plan.json`**: The reference plan. In ablation mode, this is what the agent uses. Check if steps are logical.
*   **`oracle_verify.py`**: The verification script. Check if it correctly loads your data types.

### 2. Execution Logs (Phase 2)
Located in: `{output_dir}/artifacts/`
*   **`code_iter{N}.py`**: The initial code generated by the agent in iteration N.
*   **`code_iter{N}_try{M}_refined.py`**: The refined code after attempt M failed.
*   **`execution_log_iter{N}_try{M}.txt`**: STDOUT/STDERR from running the code in iteration N, attempt M.
    *   *Look here for SyntaxErrors, ImportErrors, or RuntimeErrors for each retry.*

### 3. Verification Logs (Phase 3)
Located in: `{output_dir}/artifacts/`
*   **`verification_logs.txt`**: Output from `oracle_verify.py`.
    *   *Look here for "Shape mismatch", "Value mismatch", or "Loader Error".*

### 4. Agent "Thinking" Logs
Located in: `{output_dir}/trace.log` (or similar, depending on `RunLogger` impl)
*   Contains the raw LLM chain-of-thought.
*   Search for "code_gen_thinking" or "refine_thinking" to see why the agent wrote the code it did.

### 5. Common Error Types
*   **Loader Error**: If `oracle_verify.py` fails to load the output, check `phase0_dataset.py`'s `load_data` function.
*   **Shape Mismatch**: The code produced an array of wrong dimensions. Check if `golden_plan.json` specified the shape correctly.
*   **Timeout**: Code took too long (>600s). Check for infinite loops in `code_iter{N}.py`.
