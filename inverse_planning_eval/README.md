
# Inverse Planning Evaluation Pipeline

This repository contains a comprehensive pipeline for optimizing prompts, generating "Golden Plans" (ground truth test plans), and evaluating AI-generated plans against these golden standards.

## 📂 Project Structure

The pipeline is organized into three main modules, each triggered by a shell script:

1.  **`run_golden_gen.sh`**: Generates high-quality "Golden Plans" from Ground Truth (GT) code.
2.  **`run_optimization.sh`**: Uses TextGrad to optimize the system prompt for plan generation.
3.  **`run_eval.sh`**: Runs the full evaluation suite (Similarity + Multi-Judge ELO Tournament).

---

## 🚀 1. Golden Plan Generation (`run_golden_gen.sh`)

This module reverses-engineers a "Golden Standard" test plan from the provided ground truth source code. These plans serve as the anchor/reference for all subsequent evaluations.

-   **Script**: `run_golden_gen.sh`
-   **Python Logic**: `generate_golden_plans.py`
-   **Configuration**: `config/config_golden_gen.yaml`
    -   `golden_model`: Model used for generation (default: `claude-opus-4-5-20251101-thinking`).
    -   `output_dir`: Directory where JSON plans are saved.
    -   `tasks`: List of tasks and their corresponding GT code paths.

**Usage**:
```bash
./run_golden_gen.sh
```

---

## ⚡ 2. Prompt Optimization (`run_optimization.sh`)

This module uses **TextGrad** to iteratively optimize the system prompt used by AI agents to generate test plans. It aims to maximize the similarity between the agent's generated plan and the Golden Plan.

-   **Script**: `run_optimization.sh`
-   **Python Logic**: `prompt_optimization.py`
-   **Features**:
    -   Multi-model support (optimizes for a batch of models).
    -   Auto-clears logs (`optimization_artifacts.log`) before each run.
    -   Uses `Golden Plan` data generated in Step 1.

**Usage**:
```bash
./run_optimization.sh
```

---

## 🏆 3. Plan Evaluation (`run_eval.sh`)

This is the main evaluation engine. It assesses the quality of plans generated by various models (e.g., Gemini, Claude, DeepSeek, Qwen) against the Golden Plan.

-   **Script**: `run_eval.sh`
-   **Python Logic**: `run_eval.py`
-   **Evaluation Metrics**:
    1.  **Similarity**: Embedding Cosine Similarity (Semantic) + Structural JSON Similarity.
    2.  **ELO Tournament**: An All-vs-All round-robin tournament judged by a panel of LLMs.
        -   **Judges**: `claude-opus`, `gemini-3-pro`, `deepseek-v3.2` (Primary); `qwen3-max`, `gpt-5.2` (Reserve).
        -   **COI Avoidance**: If a model is competing, it is removed from the judge panel for that match.
        -   **Voting**: Majority vote determines the winner of each match.

**Usage**:
```bash
./run_eval.sh
```
**Outputs**:
-   `eval_results_similarity.json`: Quick similarity scores.
-   `eval_results.json`: Final results including ELO ratings and detailed match history.

## ⚙️ Configuration & Migration Guide

To deploy this pipeline on a new server, you must update the following configuration files to match your environment paths and API keys.

### 1. API Keys (`config/config2.yaml`)
Define the API keys and endpoints for all models.
```yaml
models:
  gemini-3-pro-preview:
    api_key: "YOUR_KEY"
    base_url: "YOUR_ENDPOINT"
    api_type: "openai"
  # ... other models
```

### 2. Task Paths (`config/config_task.yaml` & `config/config_golden_gen.yaml`)
**CRITICAL**: You must update the absolute paths to the Ground Truth code files.

**File**: `config/config_golden_gen.yaml` (Used for Golden Gen)
```yaml
tasks:
  spectral_ct:
    gt_code: "/path/to/your/data/spectral_ct_code.py"  # <--- UPDATE THIS
  # ... other tasks
```

**File**: `config/config_task.yaml` (Used for Optimization & Eval)
```yaml
task_io_paths:
  spectral_ct:
    gt_code: "/path/to/your/data/spectral_ct_code.py"  # <--- UPDATE THIS
  # ... other tasks
```

### 3. Source Code Paths
If you change the location of this repository (`inverse_planning_eval`), update the `base_dir` variable in the python scripts:
-   `generate_golden_plans.py`: `base_dir = "/your/new/path/inverse_planning_eval"`
-   `prompt_optimization.py`: `base_dir = "/your/new/path/inverse_planning_eval"`
-   `run_eval.py`: `base_dir = "/your/new/path/inverse_planning_eval"`

### 4. Environment Variables
If you are in a region with restricted access to Hugging Face, the scripts automatically set:
```python
os.environ["HF_ENDPOINT"] = "https://hf-mirror.com"
```
Ensure your server has internet access to this mirror or the official site.
