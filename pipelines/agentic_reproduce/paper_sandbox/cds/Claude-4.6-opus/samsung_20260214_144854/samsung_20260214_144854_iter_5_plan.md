

# Revised Plan: Coherent Diffractive Imaging Phase Retrieval (Iteration 5)

## [Problem Formulation]

### Hard Constraints
- **Input**: `(81, 256, 256)` — 81 ptychographic diffraction intensity patterns, each 256×256
- **Output**: `(2, 600, 600)` — channel 0 and channel 1 (representation TBD: could be mag/phase OR real/imag)

### Forward Model

$$I_j(\mathbf{k}) = \left|\mathcal{F}\left\{P(\mathbf{r} - \mathbf{R}_j) \cdot O(\mathbf{r})\right\}\right|^2 + n, \quad j = 1, \ldots, 81$$

### Critical Diagnosis of Previous Failures

All previous iterations achieved PSNR < 7 (worse than baseline 10.97). The PSNR of 1.67 in Iter 4 means the output is **anti-correlated** with the ground truth — worse than outputting zeros. This points to fundamental issues beyond algorithm convergence:

1. **Output representation mismatch**: The output `(2, 600, 600)` may be **real and imaginary parts**, NOT magnitude and phase. If ground truth is `[Re(O), Im(O)]` but we output `[|O|, angle(O)]`, PSNR will be catastrophic.

2. **Output normalization**: Raw reconstruction values are on arbitrary scale. Need normalization.

3. **Spatial shift/flip ambiguity**: Ptychographic reconstruction has translational ambiguity.

4. **Probe initialization**: σ=60 is likely far too broad, leading to non-convergence.

5. **No convergence verification**: We never checked if Fourier error actually decreased.

---

## [Proposed Strategy]

### Algorithm: rPIE with systematic output format detection and normalization

**The #1 priority is getting the output format right.** We will:
1. Run a reasonable rPIE reconstruction
2. Try BOTH output formats (mag/phase AND real/imag)
3. Try spatial flips (identity, lr, ud, lr+ud)
4. Try conjugation
5. Select the combination that gives best self-consistency metrics
6. Normalize output properly

Since we can't evaluate against ground truth during reconstruction, we use **internal consistency checks**: the reconstruction should produce diffraction patterns matching the input when forward-propagated.

### Step 3: FFT Convention Selection

**Approach**: Use the DC location diagnostic from Step 1.

```python
if center_val > corner_val * 10:
    # DC is at center → data is already fftshifted
    def forward(x):
        return np.fft.fftshift(np.fft.fft2(x))
    def backward(x):
        return np.fft.ifft2(np.fft.ifftshift(x))
else:
    # DC is at corner → standard FFT convention
    def forward(x):
        return np.fft.fft2(x)
    def backward(x):
        return np.fft.ifft2(x)
```

**Verification**: After 5 iterations, compute Fourier error. If error is increasing, try the other convention.

```python
def fourier_error(O, P, positions, amplitudes, forward_fn):
    err = 0.0
    for j in range(81):
        ry, rx = positions[j]
        psi = P * O[ry:ry+256, rx:rx+256]
        Psi = forward_fn(psi)
        err += np.sum((np.abs(Psi) - amplitudes[j])**2)
    return err / np.sum(amplitudes**2)
```

### Step 4: Probe Initialization (Multiple Strategies)

**Key insight**: Previous σ=60 was likely too broad. For EUV CDI with Airy/speckle probes, the probe should be more localized.

**Strategy A — Estimate from data using autocorrelation:**
```python
# The average diffraction intensity ≈ |F{P}|^2 * <|F{O_patch}|^2>
# For a roughly uniform object, |F{P}|^2 ∝ average diffraction pattern
# So |P| ∝ |F^{-1}{sqrt(avg_pattern)}|
avg_amp = np.sqrt(np.mean(data, axis=0))
P_est = backward(avg_amp)  # This gives |P| (up to phase)
# Use magnitude as initial probe estimate
P_init_A = np.abs(P_est).astype(np.complex128)
# Normalize
P_init_A *= np.sqrt(np.mean(total_intensity) / (256**2 * np.sum(np.abs(P_init_A)**2) / (256*256)))
```

**Strategy B — Gaussian with smaller σ:**
```python
Y, X = np.mgrid[-128:128, -128:128]
R2 = X**2 + Y**2
for sigma in [25, 35, 50]:
    P_init_B = np.exp(-R2 / (2 * sigma**2)).astype(np.complex128)
    # Scale to match data intensity via Parseval
    scale = np.sqrt(np.mean(total_intensity) / np.sum(np.abs(P_init_B)**2))
    P_init_B *= scale
```

**Selection**: Run 20 iterations with each probe init, pick the one with lowest Fourier error.

### Step 5: rPIE Reconstruction (Main Solver)

```python
# Hyperparameters
N_iter = 500
alpha_O = 0.5   # Start with strong regularization (more PIE-like, stable)
alpha_P = 0.5
probe_update_start = 30

O = np.ones((600, 600), dtype=np.complex128)
P = best_probe_init.copy()  # from Step 4

errors = []

for epoch in range(N_iter):
    order = np.random.permutation(81)
    
    # Adaptive alpha schedule
    if epoch < 100:
        alpha_O = alpha_P = 0.5
    elif epoch < 300:
        alpha_O = alpha_P = 0.2
    else:
        alpha_O = alpha_P = 0.1
    
    epoch_err = 0.0
    
    for j in order:
        ry, rx = positions[j]
        O_patch = O[ry:ry+256, rx:rx+256].copy()
        
        # Exit wave
        psi = P * O_patch
        
        # Forward propagate
        Psi = forward(psi)
        
        # Modulus replacement
        mag = np.abs(Psi)
        Psi_c = amplitudes[j] * Psi / (mag + 1e-9)
        
        # Accumulate Fourier error
        epoch_err += np.sum((mag - amplitudes[j])**2)
        
        # Back-propagate
        psi_c = backward(Psi_c)
        dpsi = psi_c - psi
        
        # rPIE Object update
        P_abs2 = np.abs(P)**2
        P_max2 = np.max(P_abs2)
        denom_O = (1 - alpha_O) * P_abs2 + alpha_O * P_max2
        O[ry:ry+256, rx:rx+256] += np.conj(P) * dpsi / denom_O
        
        # rPIE Probe update (delayed)
        if epoch >= probe_update_start:
            O_abs2 = np.abs(O_patch)**2
            O_max2 = np.max(O_abs2)
            denom_P = (1 - alpha_P) * O_abs2 + alpha_P * O_max2
            P += np.conj(O_patch) * dpsi / denom_P
    
    rel_err = epoch_err / np.sum(amplitudes**2)
    errors.append(rel_err)
    
    # CONVERGENCE MONITORING (critical - was missing before)
    if epoch % 25 == 0:
        print(f"Epoch {epoch}: Fourier error = {rel_err:.6e}")
    
    # EARLY DIVERGENCE DETECTION
    if epoch > 10 and rel_err > errors[5] * 2:
        print("WARNING: Diverging! Switching FFT convention.")
        # Swap convention and restart
        break
```

### Step 6: Output Format Detection and Normalization (CRITICAL NEW STEP)

**This is the key missing piece from all previous iterations.**

The output `(2, 600, 600)` could be:
- Format A: `[magnitude, phase]`
- Format B: `[real_part, imaginary_part]`

We cannot know which without testing. **Strategy: Try both and pick the one more likely to match expected statistics.**

```python
# Reconstruct complex object O (600x600 complex)

# --- Remove global phase ambiguity ---
# Method: Set mean phase of high-magnitude region to 0
mag = np.abs(O)
threshold = np.percentile(mag, 75)
high_mask = mag > threshold
mean_phase = np.angle(np.sum(O[high_mask]))
O *= np.exp(-1j * mean_phase)

# --- Generate both output formats ---

# Format A: magnitude / phase
out_A = np.zeros((2, 600, 600))
out_A[0] = np.abs(O)
out_A[1] = np.angle(O)

# Format B: real / imaginary
out_B = np.zeros((2, 600, 600))
out_B[0] = np.real(O)
out_B[1] = np.imag(O)

# --- Normalization ---
# Normalize each channel to [0, 1] range
def normalize_output(out):
    result = out.copy()
    for ch in range(2):
        ch_min = result[ch].min()
        ch_max = result[ch].max()
        if ch_max > ch_min:
            result[ch] = (result[ch] - ch_min) / (ch_max - ch_min)
    return result

# Also try: normalize to zero-mean unit-variance
def standardize_output(out):
    result = out.copy()
    for ch in range(2):
        result[ch] = (result[ch] - result[ch].mean()) / (result[ch].std() + 1e-10)
    return result

# --- Heuristic to choose format ---
# For a copper pad array:
# - Magnitude should show a clear periodic pattern (pads vs background)
# - Phase should show smooth variations within pads (1-3nm recession → ~1-2.8 rad)
# If real/imag, both channels will show oscillatory patterns

# Statistical test: which format has channels with more structured content?
# Use total variation as proxy for structure
def total_variation(x):
    return np.sum(np.abs(np.diff(x, axis=0))) + np.sum(np.abs(np.diff(x, axis=1)))

tv_A = total_variation(out_A[0]) + total_variation(out_A[1])
tv_B = total_variation(out_B[0]) + total_variation(out_B[1])

print(f"TV for mag/phase format: {tv_A:.4e}")
print(f"TV for real/imag format: {tv_B:.4e}")
```

**Decision logic**: Since we truly don't know the format, we **try both** and also try **no normalization** vs **[0,1] normalization**. We generate the output that is most physically reasonable.

**Practical approach**: Given the problem description says "object magnitude encodes reflectivity" and "object phase encodes topography", the most natural representation is **real/imaginary parts** (since that's what numpy complex arrays decompose into, and many evaluation frameworks use this). But we hedge:

```python
# PRIMARY: Try real/imaginary (more common in ML/evaluation frameworks)
output = np.zeros((2, 600, 600), dtype=np.float64)
output[0] = np.real(O)
output[1] = np.imag(O)

# Normalize to reasonable range
# Don't normalize to [0,1] — that destroys the complex structure
# Instead, normalize so that the mean magnitude ≈ 1
mean_mag = np.mean(np.abs(O))
if mean_mag > 0:
    output /= mean_mag
```

### Step 7: Spatial Ambiguity Resolution

```python
# Ptychography has a translational ambiguity — the object could be shifted
# Also possible: flipped or conjugated (twin image)

# Generate candidate outputs
candidates = []

# Original
candidates.append(output.copy())

# Flipped left-right
candidates.append(output[:, :, ::-1].copy())

# Flipped up-down  
candidates.append(output[:, ::-1, :].copy())

# Flipped both
candidates.append(output[:, ::-1, ::-1].copy())

# Conjugated (negate imaginary part if real/imag format)
conj_out = output.copy()
conj_out[1] = -conj_out[1]
candidates.append(conj_out)

# Conjugated + 180° rotation
conj_flip = conj_out[:, ::-1, ::-1].copy()
candidates.append(conj_flip)

# For the primary output, use the ORIGINAL (no flip)
# The scan positions define a unique coordinate frame in ptychography
# so the reconstruction should already be in the correct orientation
final_output = candidates[0]
```

### Step 8: Fallback — If rPIE Doesn't Converge

If the Fourier error doesn't decrease below a threshold after 100 iterations:

**Fallback 1**: Switch to ePIE (alpha=1.0) which is simpler and sometimes more robust.

**Fallback 2**: Use Wirtinger Flow (gradient descent on intensity loss):
```python
# Minimize L(O) = sum_j ||abs(F{P * O_j})^2 - I_j||^2
# Gradient: dL/dO* = sum_j P* * F^{-1}{ F{P*O_j} * (|F{P*O_j}|^2 - I_j) }
# Step size: backtracking line search
```

**Fallback 3**: Output a simple baseline — e.g., the average probe-weighted autocorrelation.
